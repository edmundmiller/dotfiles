Implement the following spec:

$ARGUMENTS

Use the `implementation-notes` skill. As you work, maintain a running `implementation-notes.html` file, or the repository's existing equivalent, capturing concise user-facing notes about:

- Design decisions where the spec is ambiguous
- Intentional deviations from the spec and why
- Tradeoffs considered and chosen
- Open questions for the user

Do not use the notes as private chain-of-thought. Keep them factual, reviewable, and useful to the user.
