Audit the active durable goal against fresh evidence, then continue work.

Optional focus:

$ARGUMENTS

Required loop:

1. Read the active goal and identify every explicit requirement.
2. Inspect current repo/session evidence: changed files, command output, logs, generated artifacts, commits, or other relevant sources.
3. Mark each requirement as verified, unverified, or blocked.
4. If anything is unverified and not genuinely blocked, take the next low-risk useful step now.
5. If validation fails, triage and fix the cause rather than reporting partial completion.
6. Only mark the goal complete after every requirement maps to fresh evidence.

Do not answer with only a plan, a status update, or user-actionable next steps if you can take the next step yourself.
