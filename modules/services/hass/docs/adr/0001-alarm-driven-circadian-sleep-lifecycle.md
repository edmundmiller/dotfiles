# Alarm-driven circadian sleep lifecycle

Home Assistant sleep routines should be driven by Edmund's next iPhone alarm when Edmund is home, rather than by a fixed bedtime or purely manual scenes, because the goal is to support a consistent circadian rhythm around the actual next wake time. The canonical lifecycle is **Winding Down → Get Ready for Bed → Good Night → Sleep**: a five-minute homeostasis check runs between 8 PM and midnight, targets six 90-minute sleep cycles by default, applies each phase once per alarm schedule, skips only the soft Winding Down phase when already late, and preserves the 10-minute get-ready and 15-minute fall-asleep buffers once the active sequence starts.

Consequences: **Get Ready for Bed**, **Good Night**, and **Good Morning** are the only HomeKit-facing manual routines; **Winding Down** and **Sleep** stay internal automation phases. The old **In Bed** / **Ignite** path and the voice-facing Sleep webhook are retired to avoid ambiguous voice routing. Phase helpers track which phases have run for the current alarm schedule, manual Get Ready for Bed / Good Night update those helpers, and Good Morning resets the schedule state. Notifications happen only at Get Ready for Bed and Good Night, initially to Edmund's phone when home; Monica notification/fallback alarm driving remains future work.
